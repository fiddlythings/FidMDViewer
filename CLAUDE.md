# Project Instructions

## No Implementation Without Permission

**NEVER** write, edit, or create code files unless the user has explicitly granted permission to implement. When a plan or task is presented, your default action is to **save it for reference only**. Before touching any source file, you MUST:

1. Ask the user: "Ready for me to implement this?"
2. Wait for an explicit "yes" or equivalent confirmation.

If the user says "no", "stop", "wait", or anything other than clear approval — do NOT proceed. Do not partially implement. Do not "start with the easy parts." Plans are plans, not instructions to execute.

## Session Memory

Always maintain `MEMORY.md` in the project root. This file tracks the current state of work, recent actions, and context needed to resume after a crash or rapid exit. Update it frequently — after completing tasks, making decisions, or reaching milestones. On session start, read `MEMORY.md` first to restore context.
